## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app broken. After any phase, you can run the app and demo what's been built.
2. **Infrastructure before features.** Phase 0 sets up everything — stack, auth, design system, routing — so all feature work in later phases is fast, clean, and consistent.
3. **Magic moment first.** The core flow — client logs in, fills in a brief, generates content that sounds like their brand — is live by the end of Phase 2. Everything else builds on top of that.
4. **Context injection is the product.** The guideline document system (tone of voice, content guidelines, copywriting framework) is what makes Content Engine different from a generic AI tool. Get this right before adding features.
5. **Admin experience is a first-class feature.** Lawrence is the second user of this product. The admin dashboard ships alongside the client experience — not as an afterthought.
6. **Review before proceeding.** Push each completed phase as a PR and let a review agent (e.g. [CodeRabbit](https://coderabbit.ai)) review it before starting the next phase. External review catches issues the coding agent won't flag.

## Agent Session Guide

### How to Use This Roadmap with Claude Code

1. **Start a phase:** Copy the phase prompt and paste it into a Claude Code session. Claude Code will read the roadmap, find the current phase, read only the listed reference sections, and start working through tasks.
2. **Read selectively:** Each phase lists its Reference sections — the specific parts of the PRD and vision doc needed for that phase. Claude Code reads only those sections, not the full documents.
3. **Let it work:** Claude Code reads the roadmap, finds the first unchecked task, implements it, marks it `[x]`, and moves to the next.
4. **One session = one phase (ideally):** Complete a full phase per session for best continuity. If you stop, Claude Code can resume from the first unchecked task.
5. **Push a PR after each phase:** When a phase is complete, Claude Code creates a branch, commits, and opens a PR. Let [CodeRabbit](https://coderabbit.ai) (free, automated) review the PR before starting the next phase.
6. **On-demand context:** If a task needs a section not in the phase references, Claude Code reads just that section on demand.

### Session Tips

- **Don't read everything:** The PRD and vision doc are large. Each phase's Reference sections tell Claude Code exactly what to read. Loading full documents wastes context window.
- **Don't skip tasks:** Tasks are ordered intentionally. Skipping creates dependency issues — earlier tasks create the foundation later tasks build on.
- **Verify after each phase:** Run the app after completing a phase to confirm everything works before moving on.
- **Review before proceeding:** Push a PR for each completed phase and let your review agent check it. Starting the next phase before reviewing the previous one means you build on top of unreviewed code.
- **Update the status line:** After completing tasks, update the header: `**Status:** X/62 tasks complete` and `**Current Phase:** Phase N — [Title]`.
